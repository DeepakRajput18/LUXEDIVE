# 🚗 LUXEDIVE - Premium Car Rental Platform

## 🚀 Quick Start (One Command)

### First Time Setup
```bash
# Clone repository
git clone <your-repo-url>
cd luxedive

# Run setup (installs everything)
npm run setup

# Update .env with your API keys
nano .env

# Start all services
npm run dev
```

That's it! Everything runs on one command.

## 📋 Available Commands

| Command | Description |
|---------|-------------|
| `npm run setup` | Initial setup (run once) |
| `npm run dev` | Start all services |
| `npm run stop` | Stop all services |
| `npm run frontend` | Start only frontend |
| `npm run backend` | Start only backend |
| `npm run supabase` | Start only Supabase |

## 🌐 Service URLs

After running `npm run dev`:

- **Frontend**: http://localhost:5173
- **Backend API**: http://localhost:5000
- **Supabase Studio**: http://localhost:54323
- **Supabase API**: http://localhost:54321

## 📊 Logs

All logs are saved in `logs/` directory:

- `logs/frontend.log` - React dev server
- `logs/backend.log` - Flask API
- `logs/360-renderer.log` - 360 Headless UI Engine
- `logs/360-worker.log` - AI generation logic
- `logs/supabase.log` - Database logs

View live logs:
```bash
tail -f logs/frontend.log
```

## 🛠️ Tech Stack

- **Frontend**: React 19 + Vite + Tailwind CSS
- **Backend**: Python Flask
- **Database**: PostgreSQL (Supabase)
- **AI**: Google Gemini
- **360° Engine**: Python + NodeJS + OpenCV + rembg + ThreeJS

## 🔑 Environment Variables
Copy `.env.example` to `.env` and update:
```bash
cp .env.example .env
```
Required API keys:

- `GEMINI_API_KEY` - Get from https://makersuite.google.com
- `SERPAPI_KEY` - Get from https://serpapi.com
- `SUPABASE_*` - Auto-generated by `supabase start`

## 🐛 Troubleshooting

Services won't start:
```bash
npm run stop
npm run dev
```

Port already in use:
```bash
# Kill processes on ports
lsof -ti:5173 | xargs kill -9  # Frontend
lsof -ti:5000 | xargs kill -9  # Backend
```

Python import errors:
```bash
cd backend
source venv/bin/activate
pip install -r requirements.txt
```

Database connection error:
```bash
supabase stop
supabase start
```
